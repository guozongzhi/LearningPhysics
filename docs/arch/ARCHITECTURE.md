# 系统架构设计文档

## 1. 架构总览
LearningPhysics采用**前后端分离**的云原生架构设计，基于领域驱动设计思想构建，支持高并发、高可用、易扩展的AI自适应学习平台。

### 1.1 核心设计原则
- **分层架构**: 清晰的职责划分，各层之间低耦合高内聚
- **异步优先**: 所有I/O操作采用异步非阻塞模式，提升系统吞吐量
- **可扩展性**: 模块化设计，支持功能灵活扩展和技术栈升级
- **云原生**: 全容器化部署，支持Kubernetes编排和弹性伸缩
- **安全可靠**: 完善的认证授权、数据加密和错误处理机制

### 2.1 前端应用层
- **技术栈**: Next.js 16 (App Router) + React 19 + TypeScript + Tailwind CSS
- **核心职责**:
  - 提供用户交互界面和沉浸式学习体验
  - 实现响应式设计，支持多端访问
  - 客户端状态管理和本地缓存
  - LaTeX公式渲染和可视化图表展示
- **核心模块**:
  - 学习主页和知识图谱展示
  - 智能测验和AI诊断功能
  - 学习笔记和文档管理
  - 学习分析和进度可视化
  - 管理员后台管理系统

### 2.2 API网关层
- **技术栈**: Nginx + Cloudflare CDN
- **核心职责**:
  - 负载均衡和流量分发
  - SSL/TLS终止和HTTPS支持
  - API限流和熔断保护
  - 静态资源缓存和CDN加速
  - WAF防护和恶意请求拦截
  - 请求日志和监控数据采集

### 2.3 后端服务层
- **技术栈**: FastAPI + Uvicorn + SQLModel + LangChain
- **分层设计**:
  1. **API路由层**: 处理HTTP请求，参数校验，响应格式化
  2. **服务层**: 封装核心业务逻辑，实现代码复用
  3. **数据访问层**: 封装数据库操作，实现ORM映射
  4. **AI引擎层**: 大模型调用、Prompt工程、错因分析算法
- **核心能力**:
  - 全异步非阻塞处理，高并发支持
  - 自动生成OpenAPI文档和类型校验
  - JWT认证和权限控制
  - 向量检索和语义相似度计算
  - 自适应学习算法和个性化推荐

### 2.4 数据存储层
- **PostgreSQL 16**: 主数据库，存储业务数据（用户、题目、答题记录等）
- **pgvector**: PostgreSQL扩展，支持向量存储和相似度检索
- **Redis**: 缓存层，存储会话信息、热点数据、接口限流计数器
- **对象存储**: 存储媒体文件（图片、视频、文档等），支持S3兼容接口

## 4. 技术选型决策

### 4.1 前端技术选型
| 技术 | 选型理由 | 替代方案 |
|------|----------|----------|
| Next.js 16 | App Router支持服务端渲染、静态生成、边缘计算，全栈能力强 | Vue + Nuxt |
| TypeScript | 类型安全，减少运行时错误，提升代码可维护性 | JavaScript |
| Tailwind CSS | 原子化CSS，开发效率高，统一设计规范 | 传统CSS/SCSS |
| Zustand | 轻量级状态管理，简单易用，性能优异 | Redux、MobX |
| Recharts + D3.js | 数据可视化能力强，支持自定义图表 | ECharts、Chart.js |
| BlockNote | 富文本编辑，支持Markdown和协同编辑 | Tiptap、Slate |

### 4.2 后端技术选型
| 技术 | 选型理由 | 替代方案 |
|------|----------|----------|
| FastAPI | 全异步，性能优异，自动生成API文档，类型安全 | Django、Flask |
| SQLModel | 结合SQLAlchemy和Pydantic，类型安全的ORM | Django ORM、Prisma |
| PostgreSQL + pgvector | 关系型数据库成熟稳定，pgvector提供向量检索能力 | MySQL + 独立向量数据库 |
| LangChain | 大模型应用开发框架，支持多模型切换和复杂工作流 | 原生OpenAI SDK |
| Pydantic | 数据验证和序列化，类型安全，开发效率高 | Marshmallow |

### 4.3 基础设施选型
| 技术 | 选型理由 | 替代方案 |
|------|----------|----------|
| Docker + Docker Compose | 容器化部署，环境一致性，运维简单 | 虚拟机部署 |
| Nginx | 高性能反向代理和负载均衡 | Traefik、HAProxy |
| Redis | 高性能缓存和会话存储 | Memcached |
| Cloudflare | CDN加速、DDoS防护、域名管理 | 阿里云CDN、腾讯云CDN |

## 6. 非功能需求

### 6.1 性能需求
- 页面加载时间 < 2s（首屏）
- API接口平均响应时间 < 500ms
- 大模型分析接口响应时间 < 10s（可异步）
- 支持1000并发用户同时在线
- 题库规模支持10万+题目，查询响应时间 < 100ms

### 6.2 可用性需求
- 系统整体可用性 ≥ 99.9%
- 故障恢复时间 < 30分钟
- 数据备份周期：每日全量备份，每小时增量备份
- 数据持久化可靠性：99.999999999%（11个9）

### 6.3 安全性需求
- 所有用户密码bcrypt加密存储，不可逆
- JWT Token有效期24小时，支持自动刷新
- 接口请求签名验证，防止重放攻击
- 用户输入严格校验，防止SQL注入和XSS攻击
- 敏感数据传输强制HTTPS加密
- 符合《网络安全法》和《个人信息保护法》要求

### 6.4 可扩展性需求
- 支持水平扩展，增加服务器节点即可提升性能
- 模块化设计，新增功能不影响现有模块
- 支持多大模型切换（OpenAI、豆包、通义千问等）
- 支持多学科扩展，不限于物理学科
